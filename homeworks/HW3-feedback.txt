Problem 8.3
   The test method in the previous problem is NOT 'exhaustive' because it
   doesn't test all the possible combinations of input values.  What you say
   about the testing types shows you don't really understand them.  I would
   respectfully suggest you go back and re-read that portion of the text.
   ...or ask ChatGPT...  :D
   In plain English:
      Black Box ~ no idea how function works, only need inputs and need to
         observe outputs are correct for those inputs
      White Box ~ you know how the function works, so that if some output is
         incorrect, you can see where the function is messing up
      Grey Box ~ a little of both; you may know how the function works, but
         you don't really need that knoweldge except in extreme cases
      Exhaustive ~ tests every possible combination of inputs, REGARDLESS of
         whether you know HOW the function works, but being mindful of the
         complete input set as well as its edge cases; YOU CAN do this with
         functions that take integers, but NOT with functions that use only
         floats or doubles as input
